# Agent operating instructions

## Mission
Build and maintain this repository from issue-driven requirements while keeping changes safe, focused, and easy to review.

## Source of truth
Use this order:
1. The active GitHub Issue
2. `docs/product.md`
3. Existing repository conventions
4. `.github/copilot-instructions.md`

## Working style
- Work on one issue at a time.
- Keep changes tightly scoped.
- Prefer simple, conventional implementations.
- Do not refactor unrelated areas.

## Boundaries
Do not change major architecture, security logic, or deployment approach unless a task explicitly requires it.

## Definition of done
A task is complete only when:
1. The issue requirements are met
2. Necessary validation or tests are added
3. The change is clearly described
